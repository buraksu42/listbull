# Research — Consumer To-Do App Gap Analysis (vs. listbull)

> Generated by Claude on 2026-05-05.
> Supplements `research.md` (Telegram-native rivals: listOK, ToBeDo, Taskobot). Focus: feature-level gap analysis between listbull's current 17-tool inventory and the broader consumer to-do landscape — what to prioritize, what to skip, what's already covered.
> Reference commit: `1f3e584` (post `update_settings`).

---

## TL;DR

1. **Recurring tasks are the #1 P1 gap.** Every serious competitor (Todoist, TickTick, Apple Reminders, Microsoft To Do, Any.do) ships RRULE-style recurrence with natural-language parsing ("every Monday at 9am", "every 2 weeks"). listbull only supports one-shot `due_at`. Without this, listbull cannot host real household routines (trash day, rent, weekly groceries).
2. **Subtasks / hierarchy is the #2 P1 gap.** Even Google Tasks has subtasks; Workflowy is built on infinite nesting. listbull's flat item model breaks the "yarın için alışveriş + sub-categories" pattern users expect.
3. **Smart Lists / saved filters (Today, Upcoming, This Week)** is the #3 P1 gap. Things 3's Today/Anytime/Someday and Apple Reminders' Smart Lists are the dominant UX paradigm for daily planning. listbull's Mini App shows lists-of-lists but no cross-list "what's due today" view.
4. **Templates and bulk-add via paste/forward** are the cheapest P1 wins. A3 (forwarded message → action items) is already speced; promote it from "future enhancement" to next sprint and add a `/template` shortcut.
5. **Grocery auto-categorization** (Apple Reminders + AnyList signature feature) is a high-impact P2 LLM win — listbull's BYOK Sonnet-4 can do it for free in the same `create_item` call. Likely the most defensible AI-native differentiator.

The verdict: listbull's Phase-3 inventory matches the *baseline* (CRUD + sharing + assignments + reminders) but misses the **planning layer** (recurring, smart lists, hierarchy) that all premium apps treat as table stakes.

## App-by-app summary

### Todoist
- **Tier model**: Free (5 projects, no reminders, Smart Quick Add); Pro $4–5/mo (300 projects, reminders, AI Task Assist, Ramble voice notes, file attachments, advanced filters); Business $6–8/mo.
- **Killer features**: best-in-class natural-language parser with full RRULE coverage ("every 1st Tuesday of the month", "every 2 weeks"); cross-platform reach (every OS); Email Assist (forward to inbox creates task with extracted dates); 80+ third-party integrations.
- **Where it loses**: free tier was crippled in Dec 2025 (5-project cap, no reminders); attachments paywalled; no built-in habit tracker / Pomodoro.
- **listbull comparison**: ✗ recurring (gap), ✗ subtasks (gap), ✗ filters/views (gap), ✓ NL parsing (parity via LLM, arguably stronger), ✓ sharing.

### TickTick
- **Tier model**: Free (unlimited tasks/lists/reminders, calendar view, habit tracker, Pomodoro); Premium $35.99/yr (~$3/mo) — multiple calendar views, durations, themes, custom filters, history.
- **Killer features**: bundles habit tracker + Pomodoro + Eisenhower matrix + calendar view; aggressive free tier (reminders included, unlike Todoist); attachments + comments free; 40+ themes.
- **Where it loses**: less polished than Things; collaboration weaker than Asana/ClickUp; iOS app slightly dated.
- **listbull comparison**: ✗ habit tracker (out of strategy), ✗ Pomodoro (out of strategy), ✗ Eisenhower matrix (gap — see §C), ✓ reminders, ✓ sharing.

### Things 3
- **Tier model**: One-time purchase per platform (Mac $49.99, iPhone $9.99, iPad $19.99). No subscription, no free tier.
- **Killer features**: Today / Upcoming / Anytime / Someday paradigm — the gold-standard daily planning UX; magazine-level visual design; Areas → Projects → To-dos → Checklist (4-level hierarchy); Quick Entry magic everywhere.
- **Where it loses**: Apple-only (no Android, no Web, no Windows); no collaboration of any kind; no recurring task UI is as fluid as Todoist's text-based; no API.
- **listbull comparison**: ✗ Today/Anytime/Someday view (P1 gap), ✗ project hierarchy (P2 — listbull's flat lists are intentional), — collaboration (listbull wins).

### Microsoft To Do
- **Tier model**: Free (full feature set, included with any Microsoft account).
- **Killer features**: My Day with intelligent task suggestions ("here's what to add today"); deep Outlook/Teams/Planner integration (flagged emails auto-appear); Wunderlist DNA (clean shared lists); free everywhere.
- **Where it loses**: shallow outside Microsoft ecosystem; weak NL parsing; no AI beyond Suggested; integrations limited to MS stack.
- **listbull comparison**: ✗ My Day "today" view (P1 gap), ✗ intelligent suggestions (P2 — LLM win), — Outlook integration (out of strategy), ✓ sharing (better — granular roles vs all-or-nothing).

### Apple Reminders
- **Tier model**: Free, bundled with iCloud.
- **Killer features**: Smart Lists with custom filters (tags, dates, locations, priority); Groceries list type with auto-categorization (Produce, Dairy, Bakery, etc.) — best-in-class for households; location reminders ("when I leave home"); Siri integration; iOS 18 calendar integration; truly free.
- **Where it loses**: Apple-only; no Android/Web; collaboration limited to iCloud users; no NL parsing on text input (Siri-only).
- **listbull comparison**: ✗ smart lists / saved filters (P1 gap), ✗ location reminders (P3 — Telegram has no geofence APIs), ✗ tags (P2 gap), **✗ grocery auto-categorization (P1 — high LLM ROI)**, ✓ sharing (better), ✓ NL (better).

### Google Tasks
- **Tier model**: Free, bundled with Gmail/Workspace.
- **Killer features**: Gmail "convert email → task" one-click; Calendar two-way sync (tasks block real time as of Nov 2025); subtasks.
- **Where it loses**: barebones — no priority, no attachments, no location reminders (deprecated), tasks in Calendar are private (not shareable), 100k task ceiling, recurrence capped at 1000 intervals; due dates only support time on supported platforms; no NL parsing.
- **listbull comparison**: ✗ Calendar two-way sync (P3 — out of strategy, Telegram is the calendar surrogate), ✗ subtasks (P1 gap), ✓ sharing (better), ✓ assignments (better — Tasks can't assign).

### Any.do
- **Tier model**: Free (unlimited tasks/subtasks, Smart Grocery Lists); Premium $4.99/mo (recurring, WhatsApp reminders, AI planning, location alerts, color tags); Family $9.99/mo (4 members); Workspace.
- **Killer features**: **WhatsApp integration — forward messages → tasks, voice reminders via WhatsApp** (listbull's closest analog in another chat app — confirms the "chat-native to-do" thesis); My Day daily-planning ritual ("Moment"); AI planning assistant; Smart Grocery Lists (free).
- **Where it loses**: free tier gates many features behind Premium; WhatsApp integration is bolt-on, not native (Any.do is a separate app); collaboration weaker than Todoist.
- **listbull comparison**: this is the **closest behavioral analog**. ✗ Smart Grocery (P1 — see Apple Reminders), ✗ daily-planning ritual (P1 — same as Things), ✓ chat-native (better — listbull *is* the chat surface, not a bolt-on), ✓ AI planning (parity via LLM).

### AnyList
- **Tier model**: Free; Complete $9.99/yr individual / $14.99/yr household — meal planning, web app, photos.
- **Killer features**: best dedicated household shopping app; Siri voice add; recipe-to-list parsing; meal calendar; photos on items; real-time household sync.
- **Where it loses**: shopping-focused — not a general task manager; no AI; Apple-leaning UX.
- **listbull comparison**: ✗ recipe import (P3 — niche), ✗ photos on items (P2 — Telegram supports this trivially), ✓ sharing (parity), — meal planning (out of strategy).

### OurGroceries
- **Tier model**: Free with ads; one-time fee for ad-free.
- **Killer features**: dead-simple shared shopping lists; best Alexa/Google Assistant integration; multi-store support.
- **Where it loses**: shopping only; no advanced features; ad-supported; no AI.
- **listbull comparison**: — out of scope (single-purpose app); the "shared list zero friction" UX is what listbull replicates inside Telegram.

### Workflowy
- **Tier model**: Free (250 bullets/month); Pro $4.99/mo unlimited.
- **Killer features**: infinite-depth bullet hierarchy; mirrors (live-copy bullets across the tree); zoom-in to any node; AI Quick Actions + Chat with Notes (added 2025); Kanban + templates + backlinks.
- **Where it loses**: not really a to-do app — it's an outliner/notes hybrid; weak reminders; no proper sharing roles.
- **listbull comparison**: ✗ hierarchy (P1 gap — but listbull should NOT match infinite depth; 1-level subtasks is enough), ✗ mirrors (P3 — too complex for the wedge), ✓ AI summarize (LLM win, parity).

### Notion (tasks, brief)
- Notion 3.0 (Sept 2025) launched **AI Agents** that can create pages/databases, run multi-step workflows up to 20 minutes, integrate with Slack/Drive/GitHub. AI autofill arrived for databases (auto-classify items, extract structured data). Notable as the direction-of-travel for "AI agents acting on lists" — but the UX is desktop-first, database-heavy, hostile to Telegram-native users. Out of scope for direct copy; useful as a benchmark for "how AI is reshaping this category" (answer: agentic, multi-step, autonomous).

**P1 notes:**
- *Recurring*: minimum viable is 5 patterns (`daily`, `weekdays`, `weekly:dow`, `monthly:dom`, `yearly`). Full iCal RRULE is overkill — match Todoist's NL parser ("every Monday", "every 2 weeks"), let LLM map to a `recurrence` JSONB column. New `recurrence` field on items + new `update_recurrence` tool.
- *Subtasks*: 1-level deep is enough (Google Tasks pattern). Add `parent_item_id` self-FK on items; expose via `create_item`'s optional `parent_item_id` arg; render indented in Mini App.
- *Smart lists*: need a "Today" view at minimum (cross-list, items where `due_at <= EOD today` OR has reminder today). New tool `get_today` + Mini App `(app)/today/page.tsx`.

## Recommended roadmap

> Ordered by impact-to-effort. Sprint A is the highest-leverage week of work; Sprint D is "nice to have".

### Sprint A — "Make listbull a real planner" (recurring + subtasks + Today view)

The single biggest gap. Fixes P1 in categories A, B, C all at once.

- **Schema change** (architect-approval required): add `recurrence JSONB` and `parent_item_id UUID` columns on `items`.
  - `recurrence` shape: `{ kind: 'daily' | 'weekdays' | 'weekly' | 'monthly' | 'yearly', interval: number, byday?: string[], bymonthday?: number[] }` — small subset of RFC-5545, NL-friendly.
- **New tool** `set_recurrence` — `{ item_id, recurrence | null }`. Cron change: when reminder fires on a recurring item, advance `due_at` to next occurrence instead of just setting `reminder_sent`.
- **Update tool** `create_item` and `update_item` to accept optional `recurrence` and `parent_item_id`.
- **New tool** `get_today` — returns all items with `due_at <= EOD(user_tz)` across all writable lists, grouped by list. (Alternative: extend `search_items` with a `due_window: 'today' | 'this_week'` filter — fewer tools, same functionality. Recommend extension.)
- **Mini App work**: new `(app)/today/page.tsx` route as the default landing for the Mini App; show today + overdue, with check-circle inline. Subtask rendering = 1-level indent on existing `<ItemRow />`.
- **Bot**: NL handler — "her pazartesi 9'da spor yap" → `create_item({ text: 'spor yap', recurrence: { kind: 'weekly', byday: ['MO'] }, due_at: <next Monday 9am> })`.

**Estimated effort**: 1 architect pass (schema + tool contracts), 2 backend days (executor + cron loop change), 2 frontend days (Today route + subtask render).

### Sprint B — "Capture without typing" (forward + voice + templates)

Three wedge-aligned input modalities. Each leans on infrastructure listbull already owns.

- **Promote A3 (forwarded message → action items)** out of "future" into shipped. Detect `forward_origin` in webhook, run a one-shot LLM call ("extract action items from this message"), batch-insert. ~1 day.
- **Voice transcription**: Telegram bot already receives voice messages. Pipe audio to OpenRouter (`openai/whisper-1` or equivalent) → text → existing LLM router. New env `OPENROUTER_TRANSCRIPTION_MODEL`. ~1 day.
- **New tool** `instantiate_template` — `{ template_name, list_id? }` — server-side library of named templates ("hafta sonu temizliği", "weekly groceries", "yeni proje kickoff") that expand to a list of items. Start with 3–5 hardcoded TR/EN templates; user-defined templates later.
- **New slash command** `/template` — list available templates + accept a name.

**Estimated effort**: 3 days total. No schema change.

### Sprint C — "AI does the boring sorting" (categorization + smart grocery + summary)

Pure-LLM features that lean on listbull's BYOK wedge. Zero schema change, all in `respond.ts` + system prompt.

- **Grocery auto-categorize**: when an item lands in a list whose name matches `^(market|alışveriş|grocery|shopping)` (or has a flag `is_grocery`), the LLM decorates the item with a category tag (Produce/Dairy/etc. — TR + EN dictionary). Render in Mini App as a section header.
- **List summarization**: `/summary <list>` slash → LLM reads all open items via `search_items`, returns a 2–3 sentence summary in user locale.
- **Auto-list routing improvement**: today the LLM picks a list by name; teach it to ask "this looks like groceries — should I put it in Market?" when no exact match exists, instead of dumping in Inbox. Pure prompt change.

**Estimated effort**: 1 day (system prompt iteration + 2 small tools).

### Sprint D — "Daily ritual + planning polish"

Lower-impact polish; ship if user demand confirms after A–C.

- **Snooze tool**: `snooze_item({ item_id, until: '+1h' | '+1d' | iso })` — wraps `update_item` semantically.
- **Quiet hours**: extend `update_settings` with `quiet_hours_start`/`quiet_hours_end`; cron suppresses DMs in window.
- **Multiple reminders per item**: new `item_reminders` table + `add_reminder` / `remove_reminder` tools. Defer until at least 5 unique users ask.
- **Tags**: add `tags TEXT[]` on items + `set_tags` tool. Defer — listbull's per-list model already covers most "tag" use cases.
- **Bulk operations** (Mini App only — multi-select + batch complete/delete/move). No new tools needed; UI work.

## What listbull SHOULDN'T copy (anti-patterns)

1. **Heavy email integration** (Todoist Email Assist, MS To Do/Outlook). listbull's premise is *Telegram is the hub, not email*. Forwarding an email to a Telegram bot is awkward; the Telegram-forward analog (A3) is the right substitution.
2. **Project hierarchy ceremony** (Things 3's Areas → Projects → To-dos → Checklists). Beautiful on macOS, hostile on a phone keyboard, contradicts AI-first capture. Stay flat: lists + 1-level subtasks max. Let the LLM do the implicit hierarchy.
3. **Per-feature paywalls for basics** (Todoist Pro for reminders, Any.do Premium for recurring). listbull's model is total free + BYOK. Recurring, attachments, multiple reminders — all should remain free; OpenRouter cost is the user's, infra cost is the operator's. Don't recreate the Todoist Free trap.
4. **Calendar two-way sync** (Google/Apple/MS). Massive engineering surface (CalDAV, OAuth scopes per provider, conflict resolution) for a feature Telegram-native users barely ask for. Defer indefinitely.
5. **Habit tracker / Pomodoro** (TickTick). Great features in a separate product. listbull is a *list assistant*, not a productivity-suite. Resist scope creep — every TickTick-style feature dilutes the wedge.
6. **Infinite-depth bullets / mirrors** (Workflowy). 1-level subtasks is right; deeper hierarchy is a different product (knowledge management).
7. **Location-based reminders** (Apple Reminders, Any.do). Telegram bots have no geofence API. Faking it via Mini App geolocation is technically possible but battery-hostile and PR-hostile. Skip.
8. **Browser extension** (Todoist, TickTick, Any.do). The Mini App is the surface; a browser extension competes with it for no clear win.
